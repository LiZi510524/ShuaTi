# 第一章题库 → 知识点速记 PDF 生成流程

> 适用场景：用户上传 Excel 题库，A 列是题目，B 列是正确答案，后面几列是选项；目标是把本章内容总结成一份“先记忆、再刷题”的知识点 PDF。

---

## 1. 先确认任务目标

本任务不是单纯把题目复制进 PDF，而是要完成三件事：

1. **从题库反推本章考点**：看题目反复考什么，而不是逐题堆答案。
2. **整理成适合记忆的结构**：框架、表格、易混点、口诀、判断题陷阱。
3. **输出 PDF 文件**：方便手机/平板直接背。

用户给出的表格规则：

- A 列：题目
- B 列：正确答案
- C 列及以后：选项

---

## 2. 文件处理原则

### 2.1 不要走弯路

不要做这些低效操作：

- 不要把 Excel 截图后 OCR。
- 不要逐题人工读表格。
- 不要直接把所有题目塞进 PDF。
- 不要只按题号总结，要按“考点”总结。
- 不要只做大纲，不做刷题提醒；用户的目标是提效刷题。

### 2.2 正确读取方式

优先使用 Python 的 `openpyxl` 或 `pandas` 读取 `.xlsx`：

```python
import pandas as pd

file_path = "/mnt/data/第一章.xlsx"
df = pd.read_excel(file_path, header=None)
```

然后按列解析：

```python
question_col = df.iloc[:, 0]   # A列：题目
answer_col = df.iloc[:, 1]     # B列：正确答案
option_cols = df.iloc[:, 2:]   # C列及以后：选项
```

---

## 3. 数据清洗流程

### 3.1 去掉空行

需要排除题目为空、答案为空的行：

```python
df = df.dropna(subset=[df.columns[0], df.columns[1]])
```

### 3.2 统一文本格式

清理多余空格、换行、全角/半角混乱：

```python
def clean_text(x):
    if pd.isna(x):
        return ""
    return str(x).replace("\n", " ").strip()
```

### 3.3 识别题型

可以根据题目关键词、答案形式、选项数量粗略分类：

- 单选题：答案多为一个字母，如 A/B/C/D。
- 多选题：答案可能是 AB、ACD、BC 等。
- 判断题：选项可能是“正确/错误”“是/否”，或答案为“对/错”。

题型识别不是为了严格统计，而是为了总结“高频题型”和“易错点”。

---

## 4. 从题库反推知识点

这是最关键的一步。不要逐题解释，而是要做“题库压缩”。

### 4.1 高频词提取

从题目、正确选项、错误选项中提取反复出现的关键词。

可以统计：

- 高频名词
- 高频动词
- 高频概念
- 反复出现的限定词，例如“根本”“本质”“核心”“首要”“决定性”“错误的是”

思路示例：

```python
from collections import Counter

text = " ".join(df.iloc[:, 0].map(clean_text).tolist())
# 可结合 jieba 分词，或手动按课程关键词归纳
```

### 4.2 按考点聚类

把题目归到几个大类，而不是题号顺序。常见结构：

1. 核心概念类  
2. 历史背景类  
3. 理论来源类  
4. 基本原则类  
5. 重要意义类  
6. 易混判断类  
7. 多选题高频组合类  

### 4.3 提炼方式

每个考点建议写成下面格式：

```markdown
### 考点名称
- 记忆重点：一句话说明核心内容。
- 高频问法：题目通常怎么问。
- 易错点：容易被哪个选项干扰。
- 刷题提醒：看到什么关键词优先联想到什么答案。
```

## 6. 生成 PDF 的技术流程

### 6.1 必须先读 PDF 技能说明

如果要创建 PDF，应先查看：

```bash
cat /home/oai/skills/pdfs/SKILL.md
```

重点注意：

- 中文字体支持。
- 页面大小、边距、换行。
- PDF 是否能正常打开。
- 输出路径必须在 `/mnt/data/` 下，方便给用户下载。

### 6.2 推荐用 HTML/CSS 转 PDF

中文 PDF 最稳的方式之一是：

1. 先生成 HTML。
2. 用合适中文字体渲染。
3. 转成 PDF。
4. 再抽查页面截图。

结构示例：

```python
html = """
<!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8">
<style>
body {
    font-family: "Noto Sans CJK SC", "Microsoft YaHei", sans-serif;
    line-height: 1.6;
    font-size: 12pt;
}
h1, h2, h3 {
    font-weight: 700;
}
.table {
    width: 100%;
    border-collapse: collapse;
}
</style>
</head>
<body>
...
</body>
</html>
"""
```

### 6.3 检查中文字体

生成前可以检查系统字体：

```bash
fc-list | grep -i "Noto\|CJK\|Source\|SimHei" | head
```

如果字体不合适，中文可能变成方框。这个坑很常见，别硬刚，先查字体。

## 7. PDF 质量检查

生成 PDF 后至少检查三件事：

### 7.1 文件是否存在

```bash
ls -lh /mnt/data/第一章知识点速记与刷题指南.pdf
```

### 7.2 页数是否合理

可以用 `pdfinfo`：

```bash
pdfinfo /mnt/data/第一章知识点速记与刷题指南.pdf
```

### 7.3 渲染抽查

把 PDF 前几页渲染成图片，查看是否：

- 中文没有乱码。
- 表格没有挤爆页面。
- 标题层级清晰。
- 内容没有大面积空白。
- 页眉页脚不遮挡正文。

示例：

```bash
mkdir -p /mnt/data/_renders_ch1
pdftoppm -png -f 1 -l 3 /mnt/data/第一章知识点速记与刷题指南.pdf /mnt/data/_renders_ch1/page
```

## 9. 下次直接复用的执行清单

下次遇到同类任务，按这个顺序来：

1. 读取 Excel，不要 OCR。
2. 确认 A 列题目、B 列答案、后面是选项。
3. 清洗空行和异常格式。
4. 统计题量、题型、答案形式。
5. 从题目中提取高频词和反复考法。
6. 把题目压缩成 5—8 个核心考点。
7. 每个考点写：记忆重点、常见问法、易错点、刷题提醒。
8. 单独整理判断题陷阱和多选题组合。
9. 生成适合背诵的 PDF，不要做成题库搬运。
10. 检查中文字体、页数、版式、渲染截图。
11. 最后给用户 PDF 下载链接。
